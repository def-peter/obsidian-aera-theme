# Aera 蓝色基调与正文组件设计规格

## 背景

Aera 1.0.0 本地发布候选已经完成正文基础、浅深色模式、桌面与移动端适配。当前视觉以墨绿色灰阶和绿色默认强调色为主；Callout、引用与多行代码块仍接近 Obsidian 默认结构。

本轮调整参考思源笔记主题的内容组件表现，但不复制其实现。目标是在 Obsidian 官方主题能力和 DOM 结构内，形成更清晰的蓝色基调、语义 Callout、纸张折角引用与 Monokai 多行代码块，同时保持 Aera 的安静、克制和中文优先阅读体验。

## 已确认决策

| 项目 | 决策 |
| --- | --- |
| 整体方向 | Quiet Aera：保留参考图特征，但降低饱和度和装饰存在感 |
| 默认强调色 | 以 `#1677FF` 为浅色模式基准，深色模式使用更亮的同系蓝 |
| 中性色 | 去除现有绿色倾向，改为冷中性灰与石墨灰 |
| Callout | 所有类型无边框、语义色正文、右下角原生大图标 |
| 引用 | Reading View 与 Live Preview 使用灰色纸张卡片和左上折角；Source Mode 只使用灰底 |
| 多行代码 | 浅色、深色均固定使用 Monokai 暗色方案 |
| 行内代码 | 不使用 Monokai；灰色背景加深一级，前景改为更浅的蓝灰 |
| 版本与发布 | 保持本地 `1.0.0` 候选；不自动 push、tag 或发布 |

## 目标

1. 将默认界面从绿色系迁移到以 `#1677FF` 为基准的蓝色系。
2. 保持用户自定义强调色、正文字体、字号和可读行宽优先。
3. 让所有 Obsidian Callout 共享统一结构，同时保留各类型官方语义颜色和图标。
4. 在 Reading View 与 Live Preview 中实现稳定的纸张折角引用样式。
5. 只让三反引号多行代码块使用 Monokai，行内代码保持正文体系。
6. 覆盖浅色、深色、Live Preview、Source Mode、Reading View、桌面和移动端。

## 非目标

- 不重新设计导航、标签页、文件树或设置界面布局。
- 不添加图片、远程字体、远程资源、JavaScript 或外部图标包。
- 不覆盖 Obsidian 的 Callout 类型颜色变量。
- 不增加代码行号、复制按钮或语言标签等代码块功能。
- 不隐藏或重排 Source Mode 的 Markdown 标记。
- 不改变用户字号、正文字体、等宽字体或可读行宽设置。

## 颜色系统

### 浅色模式

浅色模式使用雾白背景、冷中性灰层级和石墨正文。实现时沿用现有 `--color-base-*` 层级，不新增平行颜色系统。

建议基准如下，最终值允许在对比度测试中做小幅调整：

| 角色 | 建议值 |
| --- | --- |
| 主背景 | `#F8FAFC` |
| 次背景 | `#F1F4F8` |
| 边界与分隔 | `#DDE3EB` |
| 正文 | `#202936` |
| 辅助文字 | `#647184` |
| 默认强调色 | `#1677FF` |
| 高亮背景 | 克制的暖黄色透明层 |

浅色默认 accent 的 HSL 必须对应 `#1677FF`，用于控件、焦点和非文本强调。`#1677FF` 在雾白背景上的对比度不足以直接承担小号正文链接，因此文字 accent 从同一组用户 accent 变量动态降低亮度，默认解析为更深的蓝色并达到 WCAG AA。链接、标签、复选框、选择状态与焦点状态仍从用户 accent 变量派生，不锁死用户颜色。

### 深色模式

深色模式使用中性石墨背景，不带绿色偏色。默认 accent 使用更亮的同系蓝，以保证深色背景上的 AA 对比度；用户选择自定义 accent 后仍以用户设置为准。

| 角色 | 建议值 |
| --- | --- |
| 主背景 | `#17191C` |
| 次背景 | `#20242A` |
| 边界与分隔 | `#39414B` |
| 正文 | `#E7EBF0` |
| 辅助文字 | `#9AA5B4` |
| 默认强调色 | `#4096FF` |

### 语义色原则

- 界面默认强调色是蓝色，不将所有内容语义统一染蓝。
- Callout 的信息、提示、警告、错误、示例等继续使用各自类型颜色。
- 语义背景采用低透明度，标题使用完整语义色，正文使用与普通文字混合后的柔和语义色。
- 高亮继续使用暖黄色，避免形成单一蓝色界面。

## Callout 设计

### 结构

- 所有 Callout 类型取消边框，圆角保持 6–7px。
- 背景由当前 Callout 的 `--callout-color` 生成低饱和透明层。
- 标题使用完整语义色；正文使用稍柔和的同色，但必须满足可读性要求。
- 内容区保留足够右侧内边距，避免长中文、代码或表格与装饰图标重叠。

### 图标

- 复用 Obsidian 原生 `.callout-icon` 和当前类型的 `--callout-icon`，不嵌入 SVG 或图片。
- 原生图标移到右下角，放大为低透明度水印装饰。
- 折叠箭头继续位于右上角。
- Callout 收起时隐藏大图标，避免与折叠箭头和标题冲突。

### 兼容性

- 不定义 `--callout-info`、`--callout-warning` 等官方类型变量。
- 自定义 Callout 只要提供 Obsidian 标准颜色和图标变量，就自动获得同一结构。
- 桌面、窄分栏和移动端均不得出现文字与图标重叠或横向溢出。

## 折角引用设计

### Reading View

- 取消默认左侧彩色竖线。
- 使用浅灰纸张背景、细微轮廓、6px 圆角和舒适内边距。
- 通过 `::before` 与 `::after` 的 CSS 三角形构造左上折角和阴影，不使用图片。
- 引用正文使用中性灰；粗体、链接和行内代码保留自身语义层级。

### Live Preview

- 连续引用行在视觉上组成同一张灰色卡片。
- 只在第一行显示折角，不在每一行重复。
- 实现前必须用 Obsidian CLI 检查当前 1.12.7 的 CodeMirror DOM 和类名，再选择最低特异性的稳定选择器。
- 嵌套引用继续缩进，纸张折角只出现在最外层引用起始位置。

### Source Mode

- 不显示纸张折角。
- 保留清晰可见的 `>` 标记和原始 Markdown 结构。
- 仅添加低对比灰色背景与必要的文字颜色，不隐藏或重排标记。

## 代码设计

### 多行代码块

三反引号代码块在浅色、深色和三种正文视图中统一使用 Monokai：

| 语义 | 颜色 |
| --- | --- |
| 背景 | `#272822` |
| 正文 | `#F8F8F2` |
| 注释 | `#929388`（由经典 `#75715E` 提亮以达到 AA） |
| 关键字与标签 | `#FF4B8B`（由经典 `#F92672` 提亮以达到 AA） |
| 字符串 | `#E6DB74` |
| 函数 | `#A6E22E` |
| 属性与类型 | `#66D9EF` |
| 数值与常量 | `#AE81FF` |
| 强调与运算补色 | `#FD971F` |

- Reading View 的 `pre` 和 CodeMirror 的 fenced code line 分别获得块级 Monokai 变量。
- Source Mode 中围栏标记必须保持可见。
- 代码块取消现有浅色边框，保留 6px 圆角和低对比内边界。
- 长代码继续横向滚动，不强制折行。
- 不覆盖用户配置的等宽字体或字号。

### 行内代码

- 行内代码不使用 Monokai。
- 浅色模式的背景比当前灰色加深一级，文字使用比正文更浅的蓝灰。
- 深色模式使用略亮于正文背景的中性灰底和柔和浅灰文字。
- 行内代码仍通过官方变量控制；块级 Monokai 必须使用作用域选择器，不能污染行内代码。

## 实现边界

### SCSS 归属

- `src/foundations/_colors.scss`：冷中性灰阶、浅深色默认 accent、高亮背景。
- `src/components/_callouts.scss`：Callout 变量与必要的低特异性结构选择器。
- `src/editor/_quotes.scss`：三种正文视图的引用变量和折角选择器。
- `src/editor/_code.scss`：行内代码变量、Reading View 与编辑器代码块作用域。

不新增与上述职责重复的样式文件。

### 选择器政策

- 官方 CSS 变量优先；只有变量无法表达图标位置、折角和块级 Monokai 时才增加选择器。
- 新选择器必须精确限定组件和视图，并加入主题结构测试的允许集合。
- 保持禁止项：`!important`、`:has()`、远程 URL、运行时 `@import`、Source Map 引用和 JavaScript。
- 不设置 `--font-text-size`、`--file-line-width` 或 `--font-interface-theme`。

## 测试策略

所有实现按 TDD 执行：先添加失败断言并确认失败，再修改负责该行为的 SCSS。

### 自动化测试

1. 颜色基础测试验证浅深色灰阶不再包含绿色偏向，并验证默认蓝色 accent。
2. 对比度测试覆盖正文、辅助文字、默认 accent，以及新增组件中的关键前景与背景组合。
3. Callout 测试验证无边框、语义色消费、原生图标位置、折叠状态和选择器边界。
4. 引用测试验证 Reading View、Live Preview、Source Mode 的规则分离，且不使用 `:has()`。
5. 代码测试验证行内代码变量不等于 Monokai，并验证 Reading/编辑器多行代码块的完整 Monokai 颜色。
6. 主题政策测试继续拒绝未登记选择器和禁止语法。
7. 完整执行 `npm run check`，构建后的 `theme.css` 必须与 SCSS 一致。

### 真实 Obsidian QA

使用专用 Aera 测试 Vault 和 Obsidian CLI：

- 浅色与深色模式。
- Live Preview、Source Mode、Reading View。
- 桌面宽度、窄分栏和移动端模拟。
- 所有内置 Callout 类型至少检查代表性的 info、warning、error、example、quote。
- Callout 展开与收起状态。
- 单行、多行、嵌套引用。
- 无语言与有语言的 fenced code block，以及长行横向滚动。
- 行内代码、用户自定义 accent、正文字体、等宽字体、字号和可读行宽。
- `dev:errors` 和 error 级控制台输出为空。

每个视觉缺陷都必须补回归测试、确认红灯、最小修复后重新截图。

## 预览与发布边界

- 重新生成浅色、深色、三种正文视图和移动端 QA 截图。
- 重新生成根目录 `screenshot.png`，保持 `512x288`、无多余 PNG metadata、构图完整。
- 本轮继续使用版本 `1.0.0`，作为尚未对外发布的候选迭代。
- 不自动 push、创建 tag、GitHub Release、Pull Request 或提交 Community Themes。

## 验收标准

1. 默认界面在浅色与深色模式下均不再呈现绿色基调。
2. 浅色默认 accent 与 `#1677FF` 一致；深色使用可通过对比度的同系蓝。
3. 默认文字 accent 从同一组 accent 变量派生，在浅色与深色背景上达到 WCAG AA。
4. 用户自定义 accent、字体、字号和行宽继续生效。
5. 所有 Callout 无边框，标题和正文使用语义色，右下角显示低透明度原生图标，折叠状态无冲突。
6. Reading View 与 Live Preview 的引用显示单一左上折角；Source Mode 保留 `>` 且无折角。
7. fenced code block 在所有模式下呈现完整 Monokai；行内代码保持中性灰风格。
8. Callout、引用、代码块在窄分栏和移动端无溢出、遮挡或重叠。
9. 自动化测试、主题政策、WCAG 检查、真实 Obsidian 错误检查全部通过。
10. `theme.css`、QA 截图和 `screenshot.png` 与最新 SCSS 一致。
11. 工作区提交范围清晰，未执行任何外部发布操作。
